Inquiring Well

08/01/14

When I was working on fizzbuzz, I tried to implement a version of the solution that created a blank array first using fizzarray = Array.new(num of elements, "") and then pushed in "fizz" if a multiple of 3 was encountered, "buzz" if a multiple of 5 was encountered, and the original number if not a multiple of 3 or 5. I was hoping that this would save me a line of assigning "fizzbuzz" to multiples of 15, however, I encountered the issue that when I pushed a string onto a specific empty string, say calling fizzarray[0] << "fizz", all the elements of fizzarray would concatenate another "fizz" string. This was very peculiar and I had a lot of trouble googling for a solution, because I didn't have distinctive words to describe this problem - I received many results that addressed assigning values to specific elements or assigning values to every element, and none troubleshooting the involuntary assignment of values to every element when intending to assign values to a specific element. Eventually I attended office hours with this question, and even then it took a while to isolate the problem, which was that creating an array with the Array.new command caused every element to refer to the same block, "". Therefore, when I pushed a string onto a specific element, the underlying object was actually changed for every element.
